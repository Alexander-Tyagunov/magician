# mui — Versions & Material Design 2 vs 3

MUI Material UI = **React-only** component library implementing **Material Design 2**. No official Vue/Angular/Svelte port. Styling specifics only; assume TS + React lore live elsewhere.

## Version reality (verify before asserting — versions are the point)

Verified against npm + mui.com (2026-07):

| Major | First stable | Latest | Headline |
|-------|-------------|--------|----------|
| v5 | Sep 2021 | 5.18.0 | Emotion engine, `sx`, `styled`, experimental CSS vars |
| v6 | Aug 2024 | 6.5.0 | CSS variables **stable**, container queries, `applyStyles`, Grid2 stable, Pigment CSS opt-in |
| v7 | Mar 2025 | 7.3.11 | ESM+CJS `exports`, `Grid2`→`Grid`, lab→core |
| v9 | Apr 2026 | **9.2.0** (latest) | a11y/semantics overhaul, `GridLegacy` removed, system props removed |

- DON'T assume "v7 is latest" — **latest is 9.2.0**. Confirm `npm view @mui/material dist-tags` each time.
- **There is NO stable v8 of `@mui/material`.** The line jumped v7 → v9; the official guide upgrades **v7→v9** directly. Don't reference a v8 API.
- Peer dep through v9: `react`/`react-dom` `^17 || ^18 || ^19`. TS ≥ 4.9 (v7+). webpack ≥ 5. v9 raised browser targets (Chrome ≥117, FF ≥121, Safari ≥17).

## Material Design 2 vs 3

- DO state: **MUI implements Material Design 2**, including in v9. Getting-started says verbatim "Material UI supports Material Design 2."
- DON'T claim MUI ships MD3 / Material You. The experimental `@mui/material-next` package is **deprecated** ("work on this npm package has stopped"; last `6.0.0-alpha.126`). There is **no supported MD3 in MUI today**.
- DON'T invent an MD3 theme flag or component. If a project needs MD3, that's a design-token exercise on top of MD2 components, not a built-in mode.

## Styling engine — Emotion vs Pigment CSS

- **Emotion is the default engine through v9** (`@emotion/react` + `@emotion/styled` are default peer installs). This is runtime CSS-in-JS.
- styled-components engine via `@mui/styled-engine-sc` is supported but **not for SSR** — use Emotion for server-rendered apps.
- **Pigment CSS** (introduced v6, still **opt-in**): zero-runtime CSS-in-JS, build-time style extraction → RSC-compatible, smaller bundles. Separate `@pigment-css/react` package + bundler plugin.
  - DO reach for Pigment only when you need RSC/zero-runtime; DON'T assume it's the default or that Emotion is gone.
- React 18-and-below users: pin `react-is` to your React major (MUI ships `react-is@19`) via `resolutions`/`overrides`, else prop-type runtime errors.

## Theming — legacy vs CSS variables

- v5: theming via `createTheme` + `ThemeProvider`; CSS vars were experimental (`experimental_extendTheme`/`Experimental_CssVarsProvider`).
- **v6 stabilized CSS variables.** DO enable with the theme flag, not the old provider:
```js
import { createTheme, ThemeProvider } from '@mui/material/styles';
const theme = createTheme({
  cssVariables: true,                 // v6+: emits :root { --mui-* } vars
  colorSchemes: { light: true, dark: true },
});
```
- DO use `theme.applyStyles('dark', {...})` for color-mode styles (v6+); DON'T branch on `theme.palette.mode` in new code.
- DO reference generated vars via `theme.vars.palette.primary.main` (or raw `var(--mui-palette-primary-main)`). `theme.vars` is autogenerated — never assign to it.
- v9 note: with CSS variables on, the theme object no longer changes between light/dark (perf) — read `theme.vars.*` in styles; opt out with `<ThemeProvider forceThemeRerender />` only if you must.
- CSS variables fix dark-mode SSR flicker and enable unlimited named schemes beyond light/dark.

## Grid v1 vs Grid v2 (the recurring trap)

- **Grid v1 (legacy):** `<Grid container>` / `<Grid item xs={12} md={6}>`; spacing via negative margins; grows to full width.
- **Grid v2 (`Grid2`, stabilized v6):** breakpoint + `size`/`offset` props, spacing via CSS `gap`, items don't add spacing to their own box:
```jsx
<Grid container spacing={2}>
  <Grid size={{ xs: 12, md: 6 }} offset={{ md: 2 }}>…</Grid>
  <Grid size="grow">…</Grid>   {/* was `xs` / `true` */}
</Grid>
```
- Naming churn — track carefully:
  - v6: `Grid2` is stable (dropped `Unstable_` prefix); old `Grid` still present.
  - **v7:** `Grid2` → `Grid`; old flexbox-legacy `Grid` → **`GridLegacy`** (`gridLegacyClasses`, `MuiGridLegacy`).
  - **v9:** `GridLegacy` **removed** — only the `size`/`offset` Grid remains.
- DON'T write `<Grid item xs=…>` for v7+ new code; DO use `size={{…}}`.

## Migration checklist

- DO bump every MUI package together (`@mui/material`, `@mui/icons-material`, `@mui/system`, `@mui/lab`, `@mui/utils`) to the same major. **MUI X** (`@mui/x-*`) versions **independently** — don't force-align.
- DO run codemods: `npx @mui/codemod@latest <transform> <path>`, e.g. `v6.0.0/grid-v2-props`, `v6.0.0/theme-v6`, `v7.0.0/grid-props`, `v9.0.0/*`.
- v6: `LoadingButton` folded into `Button` (`loading` prop) as of 6.4; `Typography` `color` no longer a system prop → use `sx`; UMD bundle removed; IE11 dropped.
- v7: deep imports removed — `import { createTheme } from '@mui/material/styles'`, never `@mui/material/styles/createTheme`. Lab components (Alert, Autocomplete, Pagination, Rating, Skeleton, ToggleButton…) now import from `@mui/material`. `enableCssLayer` prop added.
- v9: **system props removed** from Box/Grid/Stack/Typography/Link → move to `sx`. `components`/`componentsProps` → `slots`/`slotProps` everywhere. Removed: `createMuiTheme`, `experimentalStyled`, `Hidden`, Dialog/Modal `onBackdropClick` & `disableEscapeKeyDown`. Semantic DOM changes (Stepper `<ol>/<li>`, Tabs roving tabindex) — re-check tests/selectors.
- DON'T migrate two majors blind: run app + snapshot/e2e after each hop (v7→v9 is a single documented jump but two majors of breakage).

## Cross-framework note

MUI is React-only. Framework-agnostic styling (Tailwind, Sass, vanilla-extract, Panda) is separate lore — don't duplicate. Emotion/Pigment engine choice is a MUI-internal concern; app-level Emotion usage is its own lore.
